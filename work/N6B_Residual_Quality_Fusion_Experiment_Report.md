```text
训练1 / seed 1: valid

```

本实验复用 N4 100 轮权重，固定 `quality_power=3.0`，仅在推理阶段扫描全尺度共享的残差质量融合系数：

```text
score = class_score * ((1 - quality_mix) + quality_mix * quality_score ** 3)
```

## 3. seed 1 现象分析

1. `quality_mix` 从 0 增大到 1 时，mAP50-95 总体上升，说明质量分数确实改善了高 IoU 候选排序。
2. `mix=0.25` 获得最高 Recall 0.767904，但 mAP50-95 只有 0.655656，低于 seed 1 baseline。
3. `mix=0.875` 是最接近折中的内部候选，Recall 0.761191、mAP50 0.844366，但 mAP50-95 仅 0.664536，较 QP3 低 0.011337。
4. `mix=1.0` 仍取得最高 mAP50-95、Precision 和 scallop mAP50-95。seed 1 上不存在同时接近 QP3 高 IoU AP、又显著恢复 Recall 的内部候选。
5. 从 seed 1 单独判断，N6-B 尚未打破 Recall 与 mAP50-95 的权衡，但最终结论必须等待有效 seed 0。

## 6. 最终结论

```text
N6-B residual quality fusion: failed
seed 2 validation: not required
quality_mix refinement: stopped
post-training power/mix search: stopped
current best model: N4-QP3
```

N6-B 的负结果说明 N4 的 Recall 损失并非仅由推理阶段质量因子过强造成。弱化质量因子会迅速损失高 IoU AP，却不能跨 seed 稳定恢复 Recall，因此下一步应修改训练阶段的质量监督机制。

## 7. 下一步

停止在 DUO test 上继续搜索 `quality_power` 或 `quality_mix`。下一实验先进行质量分支诊断，分别统计 P3/P4/P5 的质量预测与匹配框真实 IoU 的相关性、均值和分布，再据此设计 N6-C 训练阶段改进。优先候选为分尺度质量损失权重或正样本质量目标平滑，但必须先由诊断结果确定，不能直接叠加新的损失项。
